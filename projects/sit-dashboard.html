01 Project Overview

The SIT Student Dashboard is a single-file, browser-based web application built as a practical daily tool for students at the Singapore Institute of Technology. It consolidates timetable views, campus alerts, and quick-access links into a unified interface — without requiring any backend or login infrastructure.

Built entirely with vanilla HTML, CSS, and JavaScript, the dashboard is designed for zero-dependency deployment: it runs as a static file, is hosted on GitHub Pages, and persists data per student using namespaced localStorage keys.

05 System Context

Timetable Integration

Weekly timetable view with xSITE integration support — students view their class schedule directly within the dashboard.

Drag & Drop Panels

Customisable layout with drag-and-drop panel repositioning, letting each student arrange their workspace to preference.

Per-Student Namespacing

localStorage data is namespaced by student ID, enabling multiple users to share the same browser without data conflicts.

Hosting

Single HTML file architecture deployed on GitHub Pages — no build tools, dependencies, or backend required.

06 Technical Stack
  • Frontend: HTML5 · CSS3 (custom properties, grid, flexbox) · Vanilla JavaScript (ES6+)
  • Storage: Web Storage API (localStorage) with per-student namespacing
  • Hosting: GitHub Pages (static hosting, zero-config)
  • UI: Bootstrap Icons · Google Fonts · No external JS frameworks
07 Role & Contribution

This was a solo personal project end-to-end: I designed the information architecture, built the drag-and-drop layout system with the HTML5 Drag and Drop API, implemented the dark/light theme system using CSS custom properties, and built the per-student localStorage namespacing so the dashboard could be safely shared on lab computers.

08 Verification & Validation

Cross-Browser Testing

Manually tested layout, drag-and-drop, and theme persistence across Chrome, Safari, and Edge on desktop and mobile.

Data Isolation

Verified that namespaced localStorage correctly isolates timetable and layout data between different student IDs on the same browser.

Timezone Fix Verification

Re-tested ICS timetable parsing after fixing a timezone bug to confirm class times displayed correctly.

Usability

Iterated panel visibility and column-picker controls based on direct use during daily timetable checks.

Built out of a genuine need — existing SIT student tools were scattered across different portals. The dashboard brings timetables, alerts, and links together in one place, and served as a personal project to develop web development skills alongside the Robotics Systems degree.
